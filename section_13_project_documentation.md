РОЗДІЛ 13. Документування результатів проєкту
13.1 Значення документації

Документація є важливою складовою будь-якого програмного продукту.
Вона забезпечує:

розуміння структури системи;

можливість подальшої підтримки;

передачу проєкту іншим розробникам;

фіксацію прийнятих рішень.

13.2 Види документації в проєкті

У рамках розробки інформаційної системи бібліотеки створено:

Технічне завдання.

Опис структури бази даних.

ER-діаграму.

Опис архітектури системи.

Інструкцію користувача.

Опис розгортання системи.

Репозиторій коду на платформі GitHub.

13.3 Структура репозиторію

У репозиторії містяться:

вихідний код;

SQL-скрипти;

документація у форматі .md;

опис комітів;

історія змін.

Контроль версій здійснюється за допомогою системи Git.

13.4 Інструкція користувача

Документація для користувача містить:

опис процесу входу в систему;

роботу з каталогом книг;

процедуру видачі та повернення;

формування звітів;

опис можливих помилок.

13.5 Технічна документація

Технічна документація включає:

опис архітектури;

структуру таблиць;

опис API (за наявності);

налаштування середовища;

резервне копіювання та відновлення.

13.6 Висновок

Документування результатів проєкту забезпечує повноцінну підтримку, масштабування та подальший розвиток інформаційної системи бібліотеки.

Наявність структурованої документації підвищує якість програмного продукту та спрощує його експлуатацію.
