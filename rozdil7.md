Розділ 7. Модель даних
7.1 Мета розділу

Метою цього розділу є побудова моделі даних, яка відображає структуру бази даних і зв’язки між сутностями системи бібліотеки. 
7.2 Основні сутності 
| Сутність   | Атрибути                                                   | Опис                                  |
| ---------- | ---------------------------------------------------------- | ------------------------------------- |
| Книга      | ID, Назва, Автор, Жанр, Рік видання, Статус                | Інформація про книги в бібліотеці     |
| Користувач | ID, Ім’я, Прізвище, Роль, Email                            | Інформація про читачів та працівників |
| Позика     | ID, ID_Книги, ID_Користувача, Дата видачі, Дата повернення | Відображає видачу книг користувачам   |
| Бронювання | ID, ID_Книги, ID_Користувача, Дата бронювання              | Інформація про заброньовані книги     |
| Звіт       | ID, Тип, Дата, Дані                                        | Дані для статистики та аналітики      |
7.3 Зв’язки між сутностями

Книга → Позика: один до багатьох (книга може мати багато позик у часі).

Користувач → Позика: один до багатьох (користувач може брати багато книг).

Книга → Бронювання: один до багатьох.

Користувач → Бронювання: один до багатьох. 
Схематично (ERD ASCII): 
[Користувач] 1 ---- * [Позика] * ---- 1 [Книга]
[Користувач] 1 ---- * [Бронювання] * ---- 1 [Книга]
