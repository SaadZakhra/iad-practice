Розділ 6. Архітектура системи та залежності між підсистемами
6.1 Мета розділу

Метою цього розділу є опис архітектури інформаційної системи бібліотеки, виділення її підсистем та взаємозв’язків між ними. Це допомагає зрозуміти структуру системи та спрощує її подальшу розробку.

6.2 Архітектура системи

Система бібліотеки будується за трирівневою архітектурою:

Клієнтський рівень (Front-end)

Веб-інтерфейс для користувачів (читачів, бібліотекарів, адміністраторів).

Містить форми для пошуку книг, бронювання, видачі та обліку.

Серверний рівень (Back-end)

Обробляє запити користувачів.

Відповідає за бізнес-логіку: перевірка наявності книги, видача, бронювання, контроль термінів повернення.

Рівень бази даних (Database)

Зберігає інформацію про книги, користувачів, історію позик, бронювання.

Забезпечує безпеку та цілісність даних.

6.3 Залежності між підсистемами

Front-end взаємодіє із Back-end через API (HTTP-запити).

Back-end звертається до бази даних для отримання та збереження інформації.

Всі підсистеми разом забезпечують цілісну роботу системи.
