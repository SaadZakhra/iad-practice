Розділ 5. User Stories у UML-нотації
5.1 Мета розділу

Метою цього розділу є візуалізація функціональних сценаріїв системи у вигляді UML-діаграм прецедентів (Use Case Diagram). Це допомагає краще розуміти взаємодію користувачів із системою.

5.2 UML-нотація (Use Case Diagram) 
          +---------------------------+
          |     Інформаційна          |
          |     система бібліотеки    |
          +---------------------------+
           /       |        |        \
          /        |        |         \
     Читач     Бібліотекар  Адміністратор
      |            |             |
      |            |             |
   [Пошук книг]   [Видача книг] [Керування користувачами]
   [Бронювання]  [Прийом книг] [Генерація звітів]
   [Історія позик] Актори: Читач, Бібліотекар, Адміністратор

Прецеденти (Use Cases) відображають дії, які користувачі можуть виконувати.

В UML можна використовувати графічні редактори, наприклад draw.io, StarUML, або Lucidchart, щоб зробити професійний варіант діаграми.
