РОЗДІЛ 8. Обґрунтування вибору типу бази даних
8.1 Аналіз вимог інформаційної системи бібліотеки

Інформаційна система бібліотеки повинна забезпечувати:

зберігання даних про книги, авторів, жанри;

формат читачів;

реєстрацію видачі та повернення книг;

контроль термінів використання;

формування звітів;

забезпечення цільності та достовірності даних.

Дані мають чітку структуру та зв'язки між сутно (книга–автор, книга–жанр, читач–видача тощо).

8.2 Вибір типу бази даних

Для реалізації системи вибрано реляційну базу даних (SQL) .

Оптимальним рішенням є використання СУБД PostgreSQL .

8.3 Причини вибору реляційної бази даних
1. Наявність чітких зв'язків між даними

У бібліотечній системі таких відносин:

Один автор — багато книг (1 :N )

Одна книга — багато видач (1 :N )

Один читач — багато видач (1 :N )

Багато авторів — багато книг (M :N )

Реляційна модель дозволяє коректно реалізувати ці зв'язки через зовнішні ключі та проміжні таблиці.

2. Забезпечення цілісності даних

PostgreSQL підтримує:

первинні ключі (PRIMARY KEY);

зовнішні ключі (FOREIGN KEY);

обмеження (NOT NULL, UNIQUE);

транзакції (ACID-властивості).

Це гарантує, що:

неможливо видати книгу неіснуючому читачу;

неможливо видалити автора, якщо з ним пов'язані книги;

дані залишаються узгодженими.

3. Підтримка складних запитів

Бібліотечна система потребує:

пошук книг за автором, жанром, роком;

формування звітів;

підрахунок статистики;

фільтрації прострочених книг.

SQL дозволяє реалізувати такі запити ефективно та оптимізовано.

4. Надійність та продуктивність

PostgreSQL:

є безкоштовною та відкритою СУБД;

має високу продуктивність;

підтримує індексацію;

забезпечує резервне копіювання та відновлення.

8.4 Альтернативні варіанти

Можливі альтернативи:

NoSQL (наприклад, документні БД)

Файлове зберігання

Локальні вбудовані БД

Однак NoSQL менше підходить через складні міжтабличні зв’язки та потребу в транзакційній групі.

8.5 Висновок

Для інформаційної системи бібліотеки найбільш доцільним є використання реляційної бази даних PostgreSQL, після чого:

забезпечує цільність даних;

підтримує складні запити;

масштабується;

є надійною та безкоштовною.
