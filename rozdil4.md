Розділ 4. Функціональні сценарії (User Stories)
4.1 Мета розділу

Метою цього розділу є опис основних функціональних сценаріїв роботи користувачів із системою у вигляді User Stories. Це допомагає зрозуміти потреби користувачів та визначити пріоритети функцій. 
4.2 User Stories 
| Роль користувача | Функція                  | User Story                                                                                                |
| ---------------- | ------------------------ | --------------------------------------------------------------------------------------------------------- |
| Читач            | Пошук книги              | Як читач, я хочу шукати книги за автором, назвою або жанром, щоб знайти потрібну книгу швидко.            |
| Читач            | Бронювання книги         | Як читач, я хочу забронювати книгу онлайн, щоб бути впевненим, що вона буде доступна для мене.            |
| Читач            | Перегляд історії позик   | Як читач, я хочу бачити список книг, які я позичав, щоб контролювати терміни повернення.                  |
| Бібліотекар      | Видача книг              | Як бібліотекар, я хочу швидко оформлювати видачу книг читачам, щоб облік був актуальним.                  |
| Бібліотекар      | Прийом книг              | Як бібліотекар, я хочу реєструвати повернення книг, щоб вести точний облік.                               |
| Адміністратор    | Управління користувачами | Як адміністратор, я хочу додавати та видаляти користувачів, щоб підтримувати актуальність бази.           |
| Адміністратор    | Генерація звітів         | Як адміністратор, я хочу формувати звіти про активність системи, щоб аналізувати ефективність бібліотеки. |
